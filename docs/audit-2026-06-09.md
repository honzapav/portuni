## P0 — data loss and security

### S1. Sync can silently overwrite local edits (data loss)
- `src/domain/sync/engine.ts:412-419` — `localHashFor()` fabricates
  `last_synced_hash` from the current local hash when no `file_state` row
  exists. On a second device, restored backup, or wiped `~/.portuni/sync.db`,
  locally edited files are classified `pull` instead of `conflict`, and
  `POST /nodes/:id/sync` (`src/api/nodes.ts:420-431`) auto-overwrites them.
- `pullFile` (`engine.ts:249-293`) has no dirty check at all; `portuni_pull`
  on a file with unpushed local edits destroys them.
- Fix: when no prior `file_state` exists, persist cache fields but leave
  `last_synced_hash` NULL (unknown = conflict, never pull); in `pullFile`,
  refuse when local hash differs from `last_synced_hash` unless `force: true`
  (or write `<name>.remote-conflict` alongside).

### S2. Moving a node across organizations orphans all tracked files
- `src/domain/edges.ts:84-149` (`moveNodeToOrganization`) rebinds only the
  `belongs_to` edge. All `files.remote_path` values still start with the old
  org root, so `deriveLocalPath` throws, files show as `orphan`, and new
  stores split into a different remote subtree.
- Fix: enumerate tracked files inside the move and run the `renameFolder`
  equivalent (remote rename + batched `remote_path` UPDATE), or refuse the
  move when tracked files exist and require an explicit migration.

### S3. FK cascades never fire on local `file:` databases
- `src/infra/db.ts` never sets `PRAGMA foreign_keys = ON`; SQLite defaults
  to OFF. `portuni_delete_node` purge (`src/mcp/tools/nodes.ts:306-312`)
  relies on CASCADE for `files`/`events` — in stdio mode, desktop local
  mode, and the dev fallback the rows survive as orphans (and feed
  discovery's known-remote set forever).
- Fix: enable `foreign_keys = ON` in `getDb()`/`getLocalDb()` for file URLs,
  and delete `files`/`events` explicitly in the purge batch as belt-and-braces.

### S4. Tauri webview: `csp: null` + `withGlobalTauri: true`
- `src-tauri/tauri.conf.json:26,28`. No CSP plus the full IPC namespace on
  `window.__TAURI__` means any future XSS (markdown rendering, node names)
  escalates to full IPC: `api_request` with the stored bearer token,
  Keychain writes, `launch_claude_for_node` (spawns shell commands).
- Fix (both are one-line config changes; the code already uses module
  imports everywhere):
  - `"withGlobalTauri": false`
  - `"csp": "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; img-src 'self' data: https:; connect-src 'self'"`

## P1 — backend correctness

### B1. Auto-adopt uploads junk files; ignore matcher is dead code
- `src/shared/portuniignore.ts` (`compileIgnorePatterns`) is referenced
  nowhere. Neither `walkMirror` (`engine.ts:766-812`) nor
  `listUntrackedLocal` (`discover-local.ts:64-94`) filters dotfiles.
  Opening a mirror in Finder creates `.DS_Store`; the next sync run
  auto-adopts and uploads it to Drive.
- Fix: skip `name.startsWith(".")` + apply `.portuniignore` in both walkers.

### B2. OpenDAL adapter `list()` is non-recursive
- `src/domain/sync/opendal-adapter.ts:89-110` lists one level and skips
  directories, but `runDiscovery` expects the full subtree (Drive adapter
  walks recursively). All synced files live under `wip/` etc., so fs-remote
  discovery never surfaces `new_remote` files.
- Fix: `op.list(path, { recursive: true })`.

### B3. No Unicode normalization on file paths (Czech diacritics)
- Byte-wise path comparison in `discover-local.ts:82`, `engine.ts:787`,
  `runDiscovery`, and `storeFile` (`engine.ts:115`). APFS can return NFD
  from `readdir` while DB paths are NFC → `Příloha.md` is flagged untracked
  and re-uploaded as a duplicate. Worse: `copyFile(src, dst)` where both
  strings name the same physical file (NFC/NFD or case-insensitive match)
  can truncate the source.
- Fix: `.normalize("NFC")` at all boundaries (readdir results, caller paths,
  remote listings); guard `copyFile` with a `stat.ino` equality check.

### B4. `renameFile` leaves remote/DB inconsistent on local failure
- `engine-mutations.ts:794-804` — remote rename succeeds, then a non-ENOENT
  local rename failure throws before the DB UPDATE; file becomes `orphan`.
  `moveFile` already handles this (lines 195-239); reuse that pattern.

### B5. Missing uniqueness + check-then-insert races on `files`
- Only `idx_files_node` exists. `storeFile` and `adoptFiles` do
  SELECT-then-INSERT; concurrent writers (sidecar + tmux server + agents)
  can create two rows for one remote file.
- Fix: `CREATE UNIQUE INDEX ... ON files(node_id, remote_name, remote_path)
  WHERE remote_path IS NOT NULL` + upsert. Same class of race in the Drive
  adapter (`drive-adapter.ts:86-159`) creates duplicate Drive files/folders
  (Drive allows same-name siblings); re-query after create and dedupe.

### B6. Non-atomic multi-step writes
- `routing.ts:81-84` `replaceRules`: DELETE then per-row INSERT — crash or
  concurrent `resolveRemote` sees an empty routing table. Use `db.batch`.
- `events.ts:178-203` `portuni_supersede`: supersede then INSERT — failure
  strands the old event. Use `db.batch`.
- Migrations (`schema-migrations.ts` table rebuilds) run untransacted and
  can race when two servers share one Turso DB.

### B7. HTTP server crash/hang edges
- `src/http/server.ts:55` and `middleware.ts:225` — `new URL(req.url)` on a
  malformed request-target throws outside try/catch → unhandled rejection →
  process exit. Wrap the handler body.
- Shutdown never calls `closeIdleConnections()`; SIGINT can hang on
  keep-alive connections.

### B8. Mirror path collisions from name slugs
- `mirror-create.ts:184-199` — local mirror leaf is `slugify(node.name)`
  while remotes use the collision-free `sync_key`. Two nodes slugging to
  the same name in one org/type register the same local directory and
  cross-attribute files. Derive the local leaf from `sync_key`; reject
  `customPath` already registered to another node.

### B9. `handleSyncRun` resurrects intentionally deleted local files
- `api/nodes.ts:417-431` auto-pulls `deleted_local`. A user who deleted a
  mirror file on purpose gets it restored every sync. Surface as a decision
  instead of auto-pull.

### B10. Smaller backend correctness items
- `renameFolder` LIKE prefix not escaped (`engine-mutations.ts:334-337`) —
  `_`/`%` in folder names over/under-match. Add `ESCAPE '\'`.
- `moveFile` cross-remote repair hint claims "No state changed" even when
  `dst.put` succeeded (`engine-mutations.ts:159-187`); `local_done` always
  true. Track which phase failed.
- `inferSectionFromPath` (`engine-mutations.ts:73-77`) matches `/outputs/`
  anywhere in the path.
- Move detection (`engine.ts:655-681`) collapses identical-content files to
  one map entry; duplicate move proposals for the same `file_id`.
- `api/events.ts:95-98` doesn't validate `status` (opaque 409 from CHECK).
- `transport.ts:82-93` builds a throwaway McpServer for sessionless
  non-initialize requests; `MAX_SESSIONS` check races.
- sync.db shared by two processes without busy-timeout; `SQLITE_BUSY`
  swallowed in `cachedRemoteStat` → transient `orphan` classifications.

## P1 — frontend correctness

### F1. File create/rename/delete are dead buttons in the desktop app
- `DetailPane.tsx:474-523` still uses `window.prompt/confirm/alert`, which
  are no-ops in the Tauri macOS webview (documented in this repo's own
  comments referencing commit d229d84). In `Portuni.app`: "Nový soubor",
  "Přejmenovat", "Smazat" do nothing; all error alerts are swallowed.
- Fix: inline forms/confirm UI (pattern already used elsewhere in
  DetailPane) or the Tauri dialog plugin.

### F2. `graphError` is never cleared → permanent full-screen error overlay
- `App.tsx:101-105, 309-321, 517-529`. One transient refetch failure paints
  "Nepodařilo se načíst graf" over the main pane forever. Also
  `openSessionForNodeId` (`App.tsx:421,430`) funnels terminal-launch errors
  into the same overlay with the wrong message.
- Fix: `setGraphError(null)` on every successful fetch; separate toast for
  session-open failures.

### F3. Node-detail fetch race in graph view
- `App.tsx:138-155` (selection) and `326-338` (5s poll) have no
  cancellation/out-of-order guard (the workspace effect at 244-267 does).
  Slow response for node A can land after selecting node B and show A's
  detail under B's selection.
- Fix: same `cancelled` flag pattern; compare `selectedId` before set.

### F4. App-wide re-renders per terminal chunk / keystroke / second
- Every `pty-data` chunk (`App.tsx:361-364`), every CodeMirror keystroke
  (editor state lifted into App via `useFileEditor`, `App.tsx:289`), and a
  1-second `now` tick (`App.tsx:343-346`) re-render the entire tree —
  Sidebar, GraphView, DetailPane, terminals — none are memoized.
- Fix: throttle activity updates (~250ms), move `now` into the components
  showing the dots, wrap heavy children in `React.memo`, move editor state
  out of App.

### F5. Tauri link handling gaps
- Terminal web links (`TerminalPane.tsx:167-183`) still use the broken
  `plugin:shell|open` path that commit 1f927a1 migrated away from elsewhere;
  clicking a URL in the embedded terminal silently does nothing.
- Markdown preview links (`MarkdownPreview.tsx:14-18`) are plain
  `target="_blank"` anchors — no-ops in the webview.
- Fix: route both through `openExternal()` in `lib/backend-url.ts`.

### F6. Unsaved editor changes lost without warning
- `closeEditor` (`App.tsx:301-304`), the fullscreen X, opening another file,
  app quit, reload — none check `dirty`. Add inline confirm-on-close,
  browser `beforeunload`, and Tauri `onCloseRequested`.
- Related: pane↔fullscreen transition remounts `EditorBody`
  (`EditorPane.tsx:62-63`) — resets edit/preview mode, cursor, scroll, and
  undo history. Lift mode state; render one re-parented instance.

### F7. Event mutations swallow errors
- `DetailPane.events.tsx:28-65, 191-203` — `try/finally` with no catch on
  save/archive/resolve/add. Failed PATCH looks saved. Route through the
  existing `onError` channel.

### F8. Stale `untracked` files shown for the wrong node
- `DetailPane.tsx:258-275` reset effect clears sync state but not
  `untracked`; previous node's files render under the new node and clicking
  one opens a 404. Add `setUntracked([])`.

### F9. Smaller frontend items
- Sidebar O(orgs×nodes×edges) scan every render (`Sidebar.tsx:493-517`) —
  memoize a child-count map.
- `focus` + `visibilitychange` both fire on activation → duplicate
  refetches (`App.tsx:309-321`); debounce.
- Triple 5s polling stack while DetailPane open (sync-status + node + mcp
  info); poll sync-status only on the Files tab, skip when hidden.
- Cmd+K/Cmd+T global shortcuts steal focus from inputs/terminal
  (`Sidebar.tsx:252-263, 621-632`).
- GraphView timers can fire after `cy.destroy()` (`GraphView.tsx:1174-1223,
  2007-2013, 2091-2097`).
- `backend-url.ts:155` — `new Response(body, {status: 204})` throws;
  pass null body for 204/304.
- `pty-exit` removes the session before the exit message renders
  (`App.tsx:366-383`); plus double exit handling via `TerminalTabs.tsx:67`.
- `OwnerPicker` caches actors for the pane lifetime
  (`DetailPane.tsx:2056-2061`); EventCard edit drafts don't resync on poll
  (`DetailPane.events.tsx:22-26`).
- `SYNC_STATUS_CACHE` (`DetailPane.tsx:90`) and the history back-stack
  (`App.tsx:168-178`) grow unboundedly; `canGoBack` reads a ref in render.

## P1/P2 — performance (backend)

- `buildContextPayload` (`src/mcp/tools/context.ts:344-389`): up to 4 serial
  queries per connected node, `tableExists` re-queried per node — 100+
  Turso round-trips on the hot "start work" path. Batch with `IN (...)`,
  cache `tableExists`.
- `handlePositions` (`api/nodes.ts:568-585`): one UPDATE per node per drag
  settle. Use `db.batch`.
- `portuni_list_files` unbounded (no LIMIT) + per-row mirror lookups
  (`mcp/tools/files.ts:129-182`); `listResponsibilities` per-row
  `loadAssignees` (`responsibilities.ts:215-220`) — batched version already
  exists in `node-detail.ts:136-166`.
- `statusScan` discovery loads the whole `files` table and walks each
  mirror's full remote subtree on every status call (`engine.ts:700-712`).
  Scope the SELECT; cache remote listings like `remote_stat_cache`.
- `mirror-create.ts` regenerates all mirrors' scope configs on every new
  mirror, awaited inline on the create-node path.
- `storeFile` + Drive multipart hold ~3x file size in memory
  (`engine.ts:120`, `drive-adapter.ts:142-147`).
- `scripts/backup-turso.ts`: per-table reads are not a consistent snapshot;
  whole dump in memory.

## P2 — security hardening

- `random_token()` uses `rand::thread_rng()` (`src-tauri/src/lib.rs:258-264`)
  for the bearer token; use `OsRng` or UUIDv4.
- Drive service-account `token_uri` is unvalidated
  (`drive-config.ts:32-52`, `drive-sa-auth.ts:56`) — SSRF + signed-JWT
  exfiltration if the SA JSON is attacker-supplied. Allowlist
  `oauth2.googleapis.com` over https.
- Audit log stores absolute local paths in Turso
  (`mcp/tools/files.ts:49-51`) — privacy leak in remote/team mode.
- `devtools` feature compiled into release Tauri builds
  (`src-tauri/Cargo.toml:24`).
- `shell.open` regex allows `file:` (`tauri.conf.json:32`) — parallel path
  around the hardened `open_external` command. Drop `file`.
- No security response headers (`X-Content-Type-Options`, etc.) and
  `/mcp/info` exposes `has_auth_token` unauthenticated — acceptable on
  loopback, flag if ever exposed.

Verified strong (no action): parameterized SQL throughout, layered path
traversal guards (`safe-path.ts` + `remote-path.ts`), timing-safe token
compare, auth boot guard for non-loopback/remote-DB, token isolated in the
Rust `api_request` proxy (webview never sees it), triple scheme allowlist on
external links, Host/Origin gates, body size limit, 0600 token file writes.
